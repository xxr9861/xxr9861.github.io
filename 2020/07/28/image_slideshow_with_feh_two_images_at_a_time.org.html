Image slideshow with feh

Feh is a awesome image viewer. Here is a script, a wrapper really to see images like in a slideshow, two images at a time.

I call the script doubleview

You invoke it in a directory of images or it can alternatively take a plain text file with paths (absolute paths) of images, png or jpg.

How do you read this script?

first go to the last function that is being invoked, main()

look at the main() function, it calls other functions in a sequence.

The script (doubleview) 


#! /bin/bash
input_file="$1"
alternative_input_file='/tmp/alternative_input_file'
check_user_input(){
    case $input_file in
	-h|--h|-help|--help|-H|--H) clear; show_the_documentation ; exit ;;
    esac
    
[[ "$input_file" ]] || >"$alternative_input_file" && write_input_file && input_file="$alternative_input_file"
}
show_the_documentation(){
cat<<EOF
This script is a wrapper for feh image viewer
it shows two images at once, side by side (thumbnails really, but larger)
like in a slideshow but two images instead of one
run as
$0 
in a directory of images
or 
$0 input_file
where input_file is a file with path or realpath of images
close the window and the next window with next two images will show, till the image list is exausted.
$0 -h
to see the documentation
dependencies: feh
EOF
}
write_input_file(){
ls *.png *.jpg *.jpeg  2>/dev/null | xargs realpath
}>>"$alternative_input_file"
output_file='/tmp/images_link_file'
>$output_file
read_from_file_in_twos(){
    while read line
    do
	if [[ -z $old ]]
	then
	    #this is the first time
	    old="$line"
	else
	    #not the first time
	    view_two_image $old $line
	fi
	read junk
    done
    
}<"$input_file"
view_two_image(){
    image1="$1"
    image2="$2"
    feh -A " realpath %f   >> $output_file " -t --thumb-width 400 --thumb-height 400  $image1 $image2
}
main(){
    check_user_input
    read_from_file_in_twos
}
main
